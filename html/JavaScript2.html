JavaScript

Ya estamos más que acostumbrados a ejecutar JavaScript en nuestros navegadores, pero, ¿podemos ejecutar este código fuera del navegador?

La respuesta es que sí, gracias a numerosos entornos de ejecución basados en motores como el V8 de Chrome, que nos permiten ejecutar código JavaScript en el servidor. Estos son algunos de ellos:

Bun

Permite a los desarrolladores construir, testear, ejecutar y agrupar proyectos de TypeScript y JavaScript. Además, cuenta con un ejecutor de tests, un bundler y un gestor de paquetes compatible con Node.js.

Deno

Esta herramienta está construida sobre el motor V8 y es compatible con npm y Node.js. Deno está diseñado para soportar extensiones de JavaScript como JSX y TypeScript sin configuración extra.

La plataforma está construida para la nube, lo que le permite desplegar su aplicación utilizando Deno Deploy o cualquier otro servicio en la nube que se adapte a sus necesidades.

Node.js

Su funcionamiento se basa en realizar las operaciones de manera asíncrona, además de trabajar con un único hilo.

Node.js está diseñado para manejar múltiples conexiones concurrentes dentro de un único servidor. Cuando se realiza una operación de entrada o salida como acceder a la base de datos, se reanudarán las operaciones cuando vuelva la respuesta, en lugar de bloquear el hilo y malgastar ciclos de CPU.
