Ejecución de Código en el Cliente

Destacan tres lenguajes:

  • HTML (HyperText Markup Language): establece la estructura y el contenido de una página web.
  • Define el estilo y la presentación del contenido HTML en una página o sitio web.
  • Introduce interactividad y dinamismo en una página web. A diferencia de HTML y CSS, que son lenguajes declarativos, JavaScript es un lenguaje interpretado que se ejecuta a través del motor de JavaScript del navegador.

Capacidades de Ejecución de Código en el Navegador:

  • JavaScript y el modelo de evento: ofrece una ejecución asíncrona y la gestión eficiente de las tareas en segundo plano.
  • Manipulación dinámica del DOM: capacidad de modificar el contenido y la estructura de la página en tiempo real, permitiendo la creación de aplicaciones web interactivas y dinámicas.
  • Service Workers: capacidades avanzadas, como el soporte para aplicaciones web offline, el almacenamiento en caché de recursos, y la sincronización en segundo plano.
  • WebAssembly: Ofrece la posibilidad de ejecutar código C y C++ en el navegador a un rendimiento casi nativo.
  • APIs modernas: como WebRTC, IndexedDB, WebSockets, etc.

Limitaciones de la Ejecución de Código en el Navegador:

  • Bloqueo del renderizado: por ejemplo, si existen scripts sin defer o async.
  • Rendimiento del layout: posibilidad de desencadenarse múltiples reflows y repaints, por la realización de cambios frecuentes en el DOM.
  • Limitaciones de Seguridad: casos como Same-Origin Policy, que restringe la interacción entre recursos de diferentes orígenes para proteger al usuario, limitando algunas capacidades de comunicación entre sitios. O, el de CSP (Content Security Policy), que consisten en una política que los navegadores pueden aplicar para mitigar riesgos de inyección de scripts maliciosos (XSS), pero que puede restringir ciertas funcionalidades si no se configura adecuadamente.
  • Compatibilidad entre navegadores.
